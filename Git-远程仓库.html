git fetch
将远程主机的最新内容拉到本地暂存区
git pull
将远程主机的最新内容拉取下来后直接合并

创建一个远程仓库

  1. 新创建一个远程仓库
  2. SSH 密钥
  3. git remote add origin git@github.com:haoran-mc/Practice_git.git
  4. git push -u origin master
  5. 删除远程库：
    git remote -v
    查看远程信息；
    git remote rm <name>
    git remote rm origin ；

git push 代码到远程新分支

  • git push origin 本地分支:远程新分支

同时关联多个远程仓库

可以关联 github 的同时关联 gitee

git remote add github git@github.com:haoran-mc/Practice_git.git
git remote add gitee git@gitee.com:haoran_mc/Practice_git.git
git remote -v
查看远程库信息就能看到两个远程库；

提交的时候要表明要提交的远程库和分支

git push github master
git push gitee master

同步上游分支代码

  1. 查看所有远程库的远程地址 git remote -v
  2. 添加源分支 git remote add upstream [源项目 URL]
  3. 从源分支获取最新的代码 git fetch upstream
  4. 合并本地分支和源分支 git merge upstream/master ，或者使用 rebase
  5. Push 到 Fork 分支 git push
