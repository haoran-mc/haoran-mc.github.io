字符串哈希

字符串哈希的操作

  1. 首先把一个字符串看成一个 \(p\) 进制的数（这个 \(p\) 是自己设置的，可以取经验值）

  2. 把这个 \(p\) 进制的数转换成一个 \(10\) 进制的数（由于这个数可能会非常大，设置一个数 \(Q\)，取模）

    "ABCD"

A B C D  ->  (1 2 3 4)_p
=  (1 * p^3 + 2 * p^2 + 3 * p^3 + 4 * p^4) mod Q   // 转换成一个十进制的数，期间mod一个数Q

这样就把一个字符串转换为一个[0, Q-1]的数，

  3. 根据上面转换成10进制的方法得到所有前缀子串的哈希
    先得到每个前缀子串的哈希

    ha[0] = 0;
ha[1] = "A" 的哈希值
ha[2] = "AB" 的哈希值
ha[3] = "ABC" 的哈希值
ha[4] = "ABCD" 的哈希值
ha[5] = "ABCDE" 的哈希值
...

注意

  1. 不能把某一个字母映射为 \(0\)
    如果把 \(A\) 映射为 \(0\)，那么 \(A = (0)_{10} = 0，AA = (00)_{10} = 0\)，就会出现不同的字符串映射到同一个数，产生冲突。
  2. 哈希字符串时，可能会出现冲突。经验值是设置 \(p = 131\) 或 \(13331\) 时，把 \(Q\) 取为 \(2^{64}\)。
    这样取的话， \(99.99\%\) 的情况下不会出现冲突。

为什么要求出前缀子串的哈希

因为这样能够通过一个公式得到所有子串的哈希

                              hash?
[高位]                     <--------->          [低位]
|-------------------------|-----------|--------------|
1                         L           R
^
|----  从1开始，


我们现在已知ha[L-1]，ha[R]

ha[L-1]: 表示下面这个子串的哈希值
|------------------------|
1                      L - 1

ha[R]: 表示下面这个长的子串的哈希值
|-------------------------|-----------|
1                         L           R

让上面的两个串的高位对齐、相减得到我们想要表示出[L, R]区间的哈希值
$hash = ha[R] - ha[L] * p^{R - L + 1};$

让上面的两个串的高位对齐、相减得到我们想要表示出[L, R]区间的哈希值：\(hash = ha[R] - ha[L] * p^{R - L + 1};\)

预处理前缀哈希值: \(ha[i] = ha[i-1] * p + str[i];\)

使用unsigned long long存储

由于上面用到使用 \(Q = 2^{64}\) 取模，当ha数组使用 unsigned long long 定义，不需要进行取模运算，因为溢出即取模。
