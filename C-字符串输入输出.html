string

  • string的输入输出
    • cin遇到空格或换行停止输入
    • getline(cin, str)是iostream中一个独立的函数，遇到换行结束输入
    • cin.get()与cin.getline()是cin的成员，不能完成string类型变量的输入
    • str.c_str()将string类型转换为字符数组进行输出

差别

  • 不能接收空格与换行
    • cin
    • %s
    • cin.get()
    • cin.getline()
  • 能接收空格与换行
    • %c
    • getchar()

gets()以回车结束

注意！

  • 需要接收单个可能为空格换行的字符，用getchar()就对了
  • 字符串中包含空格的，以换行结束输入的，用gets()
    • gets会自动清理掉输入结束时的回车
    • 也就意味着如果使用的是scanf，那么需要使用getchar()吸收最后的换行符
    • 使用gets就不需要getchar()吸收最后的换行符了
  • 怎么可能会用到下面这俩，除非题目要求到某个字符结束输入
    • cin.get()
    • cin.getline()
  • 用string时，那就从cin与getline选择
  • 使用%s进行输入字符串时，不再需要&符号

不建议使用gets()

std::gets() has been deprecated in C++11 and removed in C++14. The online compiler must use C++14 or later version of the language.

More importantly, gets is known to be a security problem. Don't use it even if the compiler supports it. Use std::fgets() instead.

Unless you are required to use functions from the <cstdio> header, there are better alternatives with std::string.

std::getline()
std::istream::getline()

The links contain example code to get you started.
