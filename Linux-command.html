最初接触Linux

  1. 用户、权限、目录结构
    • 默认情况下，所有用户包括root的相关信息存储在/etc/passwd文件内，个人的密码记录在/etc/shadow中，所有的组别记录在/etc/group中
  2. 扩展：个人学习，shell的熟练使用，用户与用户权限的管理，添加系统启动时自动运行的任务，目录的划分
  3. 通过ls -al 查看文件时 每行前面10个字符代表文件权限类型
    • -rw-r–r–
    • 第一个字符代表文件的类型，d代表目录，-代表文件l代表链接文件，b代表设备文件里面的可供存储的周边设备，c代表设备文件里面的串行端口设备
    • 接下来的9个字符，每三个为一组，分别表示文件拥有者的权限，文件所属用户组的权限，其他人的权限
    • 三个一组的权限为rwx，r表示可读，w代表可写，x表示可执行，如果没有对应权限，则用-表示
  4. chgroup, chown, chmod
    • chgrp [-R] 组别名称 dirname/filename 表示更改文件所属用户组，-R表示递归更改 文件拥有者也无法使用这个命令
    • chown [-R] 用户名 账号名称[:用户组名称] 文件或目录 表示修改文件所属用户，-R表示递归更改，可选更改所属用户组 文件拥有者也无法使用这个命令
    • chmod [-R] xyz 文件或目录 表示修改文件权限
      • 有通过数字类型修改和通过字符类型修改
      • 通过数字类型修改，使用数字代表各个权限， r = 4, w = 2, x = 1， 如果有对应权限就累加，它们的和就是某个身份的权限。然后将文件拥有者，所属组别，其他人的权限按顺序组合起来就是就是xyz处的内容，例如 chmod 740 test.txt 表示文件拥有者拥有rwx权限，所属组别中用户拥有r权限，其他人没有任何权限
      • -R 表示递归
      • 文件拥有者或root可以使用chmod改变文件权限
  5. 各个权限对文件和目录的意义
    • rwx对文件的意义
      • r 读取文件内容
      • w 修改文件内容，但是不具备删除权限
      • x 执行文件的权限
    • rwx对目录的意义
      • r 读取目录结构列表，也就是能读取目录下的文件名
      • w 能够改动目录结构列表，即创建、删除、更名、移动目录下的文件
      • x 代表用户能否进入该目录成为工作目录 用户若是只有目录的r权限，没有x权限，那只能读取到目录下的文件名，无法看到其它详细信息,此时需要同时具有rx权限
  6. 简单命令
    • ls 文件与目录的查看
      • ls 文件名/目录名(可以多个) 查看对应文件(查看属性而不是内容)或对应文件夹 如果没有文件名/目录名，则默认查看当前文件夹
      • -a 全部的文件
      • -h 将文件容量以人类较易读的方式列出来
      • -l 详细信息显示 包括文件的属性和权限等数据
      • -S 以文件容量大小排序，而不是文件名
      • -t 以时间排序
      • -r 将排序结果反向输出
      • –color=never 不要根据文件特性给予颜色显示 –color=always 显示颜色 –color=auto 让系统自行依据设置来判断是否给予颜色
      • -F 根据文件、目录等信息，给予附加数据结构，例如：* 代表可执行文件 / 代表目录 = 代表socket文件 | 代表FIFO文件
      • 常用缩写 ll
    • cp 复制文件/文件夹
      • cp 源文件 目标文件
      • -p 连同文件的属性（权限、用户、时间）一起复制过去，而非使用默认属性（备份常用）
      • 在默认条件中，cp的源文件与目标文件的权限是不同的，目标文件的拥有者通常是命令操作者本身
      • 如果需要复制源文件，则需要有对源文件的r权限
      • -r 递归复制，用于目录的复制
    • rm 删除文件/文件夹
      • rm [-fir] 文件/目录
      • -f 强制删除，忽略不存在的文件，不会出现警告信息
      • -i 交互模式，在删除前会询问使用者是否操作
      • -r 递归删除，用于删除含有文件的目录，一般目录非空时不允许删除
    • rmdir 删除空的文件夹
    • mv 移动文件/文件夹
      • mv [-fiu] 源文件 目标文件
      • -f 强制移动，如果目标文件已存在，不会询问而直接覆盖
      • -i 若目标文件已经存在，就会询问是否覆盖
      • -u 若目标文件已经存在，且源文件更新，才会更新
      • cp 和 mv 命令都涉及目标文件，如果目标文件是一个文件夹，则源文件会以原来的名字存在目标文件夹，如果目标文件是一个文件名，则源文件会以目标文件名保存
    • pwd 查看当前所在目录
    • touch 修改文件时间或者创建新文件
      • 文件有三个时间，修改时间、状态时间、读取时间
      • 当文件内容修改时，修改时间(mtime, modification time)会更新; 当文件状态更新时，状态时间(ctime, status time)会更新; 当文件内容被读取时，读取时间(atime, access time)会更新
      • touch [-acdmt] 文件
      • 在默认情况下，如果touch后面指定的文件存在，则改文件的三个时间都会更新为目前的时间 如果该文件不存在，则会主动创建一个该名字的文件
      • 我们经常使用这个命令来创建新文件
    • mkdir 创建空目录
    • cat 直接查看文件内容
      • cat 文件名
      • -n 打印出行号
    • more 可翻页查看文件内容
      • 最后一行会显示目前显示的百分比，而且还可以在最后一行输入一些有用的命令
      • 空格键 代表向下翻一页
      • Enter 代表向下翻一行
      • /字符串 代表在这个显示的内容中, 向下查找字符串这个关键词
      • :f 立即显示出文件名以及目前显示的行数
      • q 结束命令
      • b 或 [ctrl]-b 代表向回翻页，不过这个操作只对文件有用，对管道无用
    • less 比more更高级的可翻页查看文件内容
      • 空格键 向下翻动一页
      • [pagedowm] 向下翻动一页
      • [pageup] 向上翻动一页
      • /字符串 向下查找字符串的功能
      • ?字符串 向上查找字符串
      • n 重复前一个查找
      • N 反向重复前一个查找
      • g 前进到这个文件内容的第一行
      • G 前进到这个文件内容的最后一行
      • q 结束命令
    • head 数据截取
      • head [-n number] 文件名
      • 如果number前面没有-字符, 截取前number行数据 如果number前面有-字符, 表示不截取后number行数据，其余截取
      • 如果没有-n参数，默认前十行
    • tail 与head反向，从后面开始截取
      • tail [-n number] 文件名
      • 如果number前面没有+字符, 截取后number行数据 如果number前面有+字符, 表示从number行开始截取
      • 如果没有-n参数，默认后十行
    • 压缩与打包
    • 后缀名 *.zip zip *.gz gzip *.bz2 bzip2 *.xz xz *.tar tar *.tar.gz tar+gz *.tar.bz2 tar+bz2 *.tar.xz tar+xz
    • gzip 可以解开zip,gzip压缩的文件
      • -c 将压缩的文件输出到屏幕上，可通过数据流重定向来处理
      • -d 解压缩
      • -v 可以显示原文件/压缩文件等信息
      • -# #表示数字，代表压缩等级，-1最快 但是压缩比差，-9最慢，但是压缩比最好，默认-6
      • 使用gzip压缩文件，原文件会删除，只保留压缩文件
      • 可以使用zcat/zmore/zless来从压缩文件查看原文件
    • bzip2 对gzip进行优化
      • -c 将压缩过程产生的数据输出大屏幕上，实际压缩数据无法输出，只是加上-d可以看到压缩文件的原文件内容，查看时不会删除原文件
      • -d -v -# 和gzip相同
      • -k 保留原始文件，而不会删除原始文件
      • 可以使用bzcat/bzmore/bzless从压缩文件查看原文件
    • xz 对bzip2进行优化
      • -d -k -c -#和bzip2相同
      • -l 列出压缩文件的相关信息
      • 可以使用xzcat/xzmore/xzless从压缩文件查看原文件
    • tar 打包命令 可以将多个文件或目录打包成一个文件，可以同时配合gzip,bzip2,xz来使用
      • gzip,bzip2,xz在压缩文件时，只能把单一文件压缩成单一压缩文件
      • -c 建立打包文件 -t 查看打包内容，重点在文件名 -x 解包 -c -t -x 不能同时出现在一条命令中 可配合-z|-j|-J使用
      • -f 后面立即接要处理的文件名，可以是要打包成的文件名，也可以是要查询的压缩文件，也可以是要解压缩的压缩文件
      • -z 使用gzip -j 使用bzip2 -J使用xz 三者不能同时出现在同一条命令中
      • -v 在压缩/解压缩过程中，将正在处理的文件名显示出来
      • -C 后面接要目录名 表示在特定目录解包 配合-x使用
      • 常用
        • 打包压缩 tar -jcv -f filename.tar.bz2 要被压缩的文件或目录名称
        • 查看 tar -jtv -f filename.tar.bz2
        • 解压缩解包 tar -jxv -f filename.tar.bz2 -C 欲解压缩到的目录
      • 打包后的文件保留的是相对路径，而不是原来的绝对路径，在解压缩解包到对应目录后，会以对应目录为基目录 可以通过tar -jtv -f filename.tar.bz2发现
      • tar命令打包时需要自己指定最终文件名，文件名最好按照可读性的需要建立对应后缀

  7. 用户与用户组
    • linux识别用户和用户组是通过UID与GID（即User ID 和 Group ID）
    • 每个文件都是直接存储对应的UID与GID，当要查看属性时，会去/etc/passwd和/etc/group查到对应的用户名和用户组名
    • /etc/passwd 记录了系统所有用户用户的信息
      • 每个用户信息占一行 thesky:x:1000:1000:thesky,,,:/home/thesky:/bin/bash
      • 共有7个字段，通过:分隔 第一个字段表示用户名，第二个字段表示密码，由于安全性的问题，实际密码加密存储在/etc/shadow，这里只保留x字符，第三个字段为用户的UID，第四个字段为用户的初始用户组GID，第五个为用户信息说明栏，没什么用，第六个字段为用户的家目录，用户登录后即进入这个目录，第七个是用户登录后获得的shell
      • 当第七个字段为/sbin/nologin时，表示该用户登录时无法获得shell环境（应该就是该用户无法用于登录）
    • /etc/shadow 保存有用户的密码，以及用户密码实效时间等相关属性
      • 每个用户密码信息占一行 avahi:uewie:18295:0:99999:7:::
      • 共有9个字段，通过:分隔 第一个字段表示用户名，第二个字段表示加密了的密码
      • 第三个字段表示最近修改密码的日期，用一个数字表示，从1970-01-01为第一天开始累计
      • 第四个字段表示密码不可被修改的天数，表示在最近修改密码后要几天才可以再次修改密码，防止用户频繁修改密码
      • 第五个字段表示密码需要重新修改的天数，表示在用户最近修改密码后，多少天内需要再次修改密码，否则密码会进入密码过期特性
      • 第六个字段表示密码需要修改期限前的警告天数，表示在密码修改期限前多少天开始警告用户修改密码
      • 第七个字段表示密码过期后的账号宽限时间，在密码进入密码过期特性状态时，会有一定的宽限天数，用户在这段时间登录，系统会强制要求修改密码，如果在这段时间没有修改密码，密码会进入失效状态，此时用户无法再修改或使用这个密码
      • 第8个字段表示账号失效日期，也是从1970-01-01累计，过了这个日期，账号将无法使用，应该也就是密码失效的意思
      • 第九个字段保留
    • /etc/group 保存用户组信息
      • 每个用户组信息占一行 test:x:1001:
      • 共有4个字段 第一个字段为用户组名，第二个字段为用户组密码，给用户组管理员使用，实际密码加密存储在/etc/gshadow,实际上已经不怎么使用用户组管理员，第三个字段表示该用户组的GID，第四个字段表示该用户组支持的账号名称，如果该用户组是某用户的初始用户组，该用户现在并不会添加到第四字段
    • 用户组
      • 初始用户组与有效用户组
        • 当用户登录时，会根据/etc/passwd中用户初始拥护组GID获取初始用户组，用户的非初始用户组（即次要用户组）可以在/etc/group找到
        • 用户支持的用户组也就是用户的初始用户组加上/etc/group中用户加入的次要用户组，可以通过groups查看，此时出现的第一个用户组名也是用户的有效用户组
        • 有效用户组是指用户在创建新文件或者新文件夹等操作时将会指定的用户组
        • 使用newgrp [用户加入的一个用户组名] 可以将用户的有效用户组切换到另一个用户组，不过这个切换实际是开启了另一个shell，在那个shell中，用户的有效用户组发生了改变，因此使用完后记得使用exit退出
    • /etc/gshadow 与用户组密码有关，现在实际已经不怎么使用用户组管理员，也就不怎么使用这个文件

sudo apt-get check 检查是否有损坏的依赖

apt-get，是一条linux命令，适用于deb包管理式的操作系统，主要用于自动从互联网的软件仓库中搜索、安装、升级、卸载软件或操作系统

  • apt命令用法
    packagename指代为软件包的名称
  • apt-get update
    在修改/etc/apt/sources.list或/etc/apt/preferences之后运行该命令。此外您需要定期运行这一命令以确保您的软件包列表是最新的
  • apt-get install packagename
    安装一个新软件包（参见下文的aptitude）

  • apt-get remove packagename
    卸载一个已安装的软件包（保留配置文档）

    apt-get remove –purge packagename
    卸载一个已安装的软件包（删除配置文档）

    apt-get autoremove packagename
    删除包及其依赖的软件包

    apt-get autoremove –purge packagname
    删除包及其依赖的软件包+配置文件，比上面的要删除的彻底一点

    dpkg –force-all –purge packagename
    有些软件很难卸载，而且还阻止了别的软件的应用，就能够用这个，但是有点冒险

    apt-get autoclean
    apt会把已装或已卸的软件都备份在硬盘上，所以假如需要空间的话，能够让这个命令来删除您已卸载掉的软件的备份

    apt-get clean
    这个命令会把安装的软件的备份也删除，但是这样不会影响软件的使用

  • apt-get upgrade
    可以使用这条命令更新软件包，apt-get upgrade不仅可以从相同版本号的发布版中更新软件包，也可以从新版本号的发布版中更新软件包，尽管实现后一种更新的推荐命令为apt-get dist-upgrade
    在运行apt-get upgrade命令时加上-u选项很有用（即：apt-get -u upgrade)。这个选项让APT显示完整的可更新软件包列表。不加这个选项，你就只能盲目地更新。APT会下载每个软件包的最新更新版本，然后以合理的次序安装它们。注意在运行该命令前应先运行 apt-get update更新数据库，更新任何已安装的软件包

    apt-get dist-upgrade
    将系统升级到新版本

    apt-cache search string
    在软件包列表中搜索字符串

    dpkg -l package-name-pattern列出任何和模式相匹配的软件包。假如您不知道软件包的全名，您能够使用“*package-name-pattern*”

    aptitude
    周详查看已安装或可用的软件包。和apt-get类似，aptitude能够通过命令行方式调用，但仅限于某些命令——最常见的有安装和卸载命令
    由于aptitude比apt-get了解更多信息，能够说他更适合用来进行安装和卸载

    显示软件包信息

    apt-cache dumpavail打印可用软件包列表

    apt-cache show pkgs
    显示软件包记录，类似于dpkg –print-avail。

    apt-cache pkgnames
    打印软件包列表中任何软件包的名称
