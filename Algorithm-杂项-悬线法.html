内容

悬线就是一根竖线，这条竖线有初始位置与高度两个性质，可以在其上端点不超过当前位置的矩形高度的情况下左右移动。

悬线法能做的事单调栈都能做，单调栈能做的事悬线法不一定能做。既然如此还为什么学习悬线法呢？因为悬线法更容易理解，有些题目用悬线法做代码更容易写。

直接上题：

Largest Rectangle in Histogram

这题在前面的 单调栈 部分讲过用单调栈的方法解决，这里使用悬线法。

思路还是寻找每个矩形左右两边第一个比自己矮的矩形。

悬线法的操作

  1. 创建两个数组 \(l, r\)，\(l[i]\) 表示下标为 \(i\) 的矩形左侧第一个低于自己的下标；\(r[i]\) 表示下标为 \(i\) 的矩形右侧第一个低于自己的下标。
  2. 初始化 \(l[i] = r[i] = i\).
  3. 遍历每个矩形，如果 \(h[i] <= h[l[i]-1]\)，需要继续向左找比自己小的矩形

悬线法的巧妙就在赋值 \(l[i]\) 时，使用 \(l[i] = l[l[i] - 1]\).

\(l[l[i] - 1]: l[i] - 1\) 左侧第一个小于自己的矩形

既然 \(l[i] - 1\) 这个位置的矩形比自己大，在遇到左侧第一个小于 \(h[l[i] - 1]\) 之前所遇到的矩形明显一定也大于 \(h[i]\)。

  1. 遍历每个数，如果 \(h[i] <= h[r[i] + 1]\)，需要继续向右找比自己小的数。
  2. 遍历每个数，包含当前这个矩形的最大矩形就是 \((r[i] - l[i] + 1) \times h[i]\)。

AC代码：

#include <cstdio>
#include <algorithm>
using std::max;
const int maxn = 1e5 + 5;
int h[maxn];
// l[i]: 表示i位置的悬线能达到的最左边的位置
// r[i]: 表示i位置的悬线能达到的最右边的位置
int l[maxn], r[maxn];
long long res;

int main() {
    int n;
    while (scanf("%d", &n) && n) {
        res = 0;
        for (int i = 1; i <= n; i++) {
            scanf("%d", &h[i]);
            l[i] = r[i] = i;
        }

        // 得到这根线向左能达到的位置
        for (int i = 1; i <= n; i++)
            // 首先不能越界，而且要比我高这根线才能继续向左前进
            while (l[i] > 1 && h[i] <= h[l[i] - 1])
                // 比我高的线都能通过，我也能
                l[i] = l[l[i] - 1];

        // 得到这根线向右能达到的位置
        for (int i = n; i >= 1; i--)
            while (r[i] < n && h[i] <= h[r[i] + 1])
                r[i] = r[r[i] + 1];

        // 计算所有矩形面积
        for (int i = 1; i <= n; i++)
            res = max(res, (long long)(r[i] - l[i] + 1) * h[i]);
        printf("%lld\n", res);
    }
    return 0;
}
