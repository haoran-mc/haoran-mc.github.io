html,body{height:100%;}

大家经常能在别人的代码中看到 html, body {height: 100%;} 这里height属性用到了百分号的形式，这个属性用于定义元素的高度，在内容区外面可以增加内边距、边框和外边距，这个属性还有其他可能的值

值 描述
auto 默认值，浏览器会计算出实际的高度
length 使用px, cm等单位定义高度
% 基于包含它的块级对象的百分比高度
inherit 规定从父元素继承height属性的值

那么这串代码的作用显而易见就是设置网页的页面大小等于浏览器的大小，而对于有些界面，删除前面的html后发现整个页面没有变化，那么 html, body {height: 100%}; 与 body {height: 100%}; 有什么区别？ 

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <title></title>
    <style>
      div {
        height: 100%;
        background-color: #ff0;
      }
    </style>
  </head>
  <body>
    <div></div>
  </body>
</html>

我们先看如果不设置html与body的height元素，这里给div设置高度为100%，然而上面提到height值为%格式时是基于包含它的块级对象的百分比高度的。也就是div的父级body的高度的百分比，然而我们并没有为body设置高度，自然就无法显示div块 

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <title></title>
    <style>
      body {
        height: 100%;
      }
      div {
        height: 100%;
        background-color: #ff0;
      }
    </style>
  </head>
  <body>
    <div></div>
  </body>
</html>

加入了body的高度后发现网页中还是什么都没有显示，说明body的百分比高度也是基于它的父级html的高度的。 

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <title></title>
    <style>
      html,
      body {
        height: 100%;
      }
      div {
        height: 100%;
        background-color: #ff0;
      }
    </style>
  </head>
  <body>
    <div></div>
  </body>
</html>

再加入html{height: 100%;}后网页中的div块就能够显示出来了，这里设置了html{height: 100%;}就相当于设置了html块的高度等于浏览器的高度。html块的高度有了，自然body的height: 100%与div的height: 100%也就能显示出来。

取消默认的内外边距

在上面显示了div块的图片中看到，我们明明设置的是100%，但侧边却有滚动条，按理说100%正好填满整个页面，这里却有多余的部分。那是因为默认的body有8px的外边距，除了body外其他许多元素也有默认的内外边距，所以一般为了去除这些默认值，使用`*{margin: 0; padding: 0;}`这串代码重置内外边距为0，这里的*是通配符，指所有的标签。 

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <title></title>
    <style>
      * {
        margin: 0;
        padding: 0;
      }
      html,
      body {
        height: 100%;
      }
      div {
        height: 100%;
        background-color: #ff0;
      }
    </style>
  </head>
  <body>
    <div></div>
  </body>
</html>

这样就没有滚动条，而且页面全部都是div块的背景色了

不设置html的相对大小

那么什么情况下删除html的高度100%不会对页面有影响呢？ 

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <title></title>
    <style>
      div {
        height: 50px;
        background-color: #ff0;
      }
    </style>
  </head>
  <body>
    <div></div>
  </body>
</html>

这里div块的height的值是用50px设置的固定大小，而之前的div块的height的值是%设置的百分比，需要参考上级元素的大小，然而如果上级元素高度没有设置，则高度百分比不起作用。如果上级元素高度的值也是百分比，那么就继续向上查找一个确定的高度。 在浏览器中，如果一个块没有设置高度与宽度，那么浏览器会为它分配最大可用的宽度，但是不分配高度，高度的大小是由子级元素的高度撑起来的。 所以这个时候div的高度是50px，虽然body没有设置，但是它的子级元素将它的高度撑到50px，html的高度也被撑到body的高度加上body的上下边距66px。

这个时候网页的背景色是默认白色的!

浏览器的背景色是body的背景色

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <title></title>
    <style>
      body {
        background: navy;
        border: 2px solid lightgreen;
      }
      div {
        background: yellow;
        margin: 10px;
        height: 50px;
        border: 2px solid lightgreen;
      }
    </style>
  </head>
  <body>
    <span style="color: red">这里是body</span>
    <div>这里是div</div>
  </body>
</html>

设置了body的背景色后发现，body的高度是10px，除了body的部分也变成了body的颜色 

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <title></title>
    <style>
      html {
        background: darkorange;
      }
      body {
        background: navy;
        border: 2px solid lightgreen;
      }
      div {
        background: yellow;
        margin: 10px;
        height: 50px;
        border: 2px solid lightgreen;
      }
    </style>
  </head>
  <body>
    <span style="color: red">这里是body</span>
    <div>这里是div</div>
  </body>
</html>

再次设置html的背景颜色，发现虽然html的区域外的颜色也变成了html的背景颜色。查阅资料后发现，这个区域确实不是html的区域，这里的是浏览器的颜色。浏览器的背景色默认是白色，浏览器会优先查找html的背景色有没有设置，如果设置了，那么浏览器的背景色就会更新为html的背景色；如果html标签的背景色没有被设置，会向下查询body的背景色，同样，如果body标签的背景色被设置了，浏览器的背景色就会更新为body的背景色，否则浏览器使用默认的白色。
