Dijkstra

Dijkstra算法基于贪心算法

Dijkstra算法的局限性

  1. 只适用于所有边的长度都是非负数的图

对于无向图，只需在邻接矩阵和邻接表上作变动，把无向边当成两条指向相反的有向边

Bellman-Ford

  • Dijkstra不能解决含有负权边图的最短路径问题
  • 使用Bellman-Ford算法(BF算法)处理负权边的问题
A  |\                          A  |\                           A  |\
   |  \                           |  \                            |  \    
   |    \                         |    \                          |    \    
   |      \  2                    |      \  2                     |      \  2 
-4 |        \                  -4 |        \                   -4 |        \
   |          \                   |          \                    |          \  
   |     2      \                 |     5      \                  |      1     \ 
B  | ------------ \ C          B  | ------------ \ C           B  | ------------ \ C

        零环                           正环                            负环

在上面负环中，从C到B最短距离是-2，使用Dijkstra算法会比较CB与CA后认为最短路径为CB，得到错误答案

  • 与Dijkstra算法相同，Bellman-Ford算法设置一个数组，用于存放从源点到达各个顶点的最短距离
  • 如果图中出现负环，在进行多次松弛之后仍能继续松弛，直到负无穷
  • 此处Bellman-Ford是发现是否有负环，而非找到最短路径
  • 时间复杂度O(VE)

Bellman-Ford找负环:

  • Bellman-Ford迭代k次意味着不超过k条边的最短路径距离
  • 所以如果迭代n次，就意味着这里存在从源点到第n个点之间的最短路径上有 n条边，而如果有n条边，根据抽屉原理，这个路径上一定有环，也一定是负环。如果无法迭代n次，就说明没有负环
  • 但是我们不用Bellman-Ford算法找负环，因为时间复杂度，我们使用spfa找负环

SPFA

  • Bellman-Ford的时间复杂度达到了O(VE)，时间复杂度非常高
  • 注意到Bellman-Ford算法每轮操作都需要遍历所有的边
  • 显然许多都是无意义的
  • SPFA(Shortest Path Faster Algorithm)
  • 将会松弛的顶点加入到队列中，只是处理这些顶点，如果没有负环的话，时间复杂度会降到O(kE)
  • 但是如果有源点到达的负环的话，时间复杂度会退化到O(VE)

Floyd

  • 如果存在顶点k，使得以k作为中介点时顶点i和顶点j的当前最短距离缩短，则使用顶点k作为顶点i和顶点j的中介点
    • if dis[i]\[k] + dis[k]\[j] < dis[i]\[j]
    • dis[i]\[j] = dis[i]\[k] + dis[k]\[j]
  • 时间复杂度O(n^3)
  • 时间复杂度决定了顶点数n被限制在200以内，因此使用邻接表来实现Floyd算法是合适且方便的
