《THE_LITTLE_SCHEMER》

car

the first S-expression.

The primitive car is defined only for non-empty lists.

cdr

the list without the car part.

The primitive cdr is defined only for non-empty lists. The cdr of any non-empty list is always another list.

cons

adds any S-expression to the front of a list.

The primitive cons takes two arguments. The second argument to cons must be a list. The result is a list.

null?

(null? s) is just another way to ask "Is s is an null list".

The primitive null? is defined only for lists.

atom?
(atom? s) is just another way to ask "Is s is an atom?".
eq?

(eq? a1 a2) is just another way to ask, "Are a1 and a2 the same non-numeric atom?".

The primitive eq? takes two arguments. Each must be a non-numeric atom.

quote
(quote ()) is a notation for the null list. Also () and '().
lat?
(lat? l) is just another way to ask "Whether every S-expression is l is an atom?".
cond
(cond …) asks questions.
lambda
(lambda …) creates a function. and
define
(define …) gives it a name.
else
(else #f), else asks if else is true. If else is true - as it always is - then the answer is #f - false.
or
(or …) asks two questions, one at a time. If the first one is true it stops and answers true. Otherwise it asks the second question and answers with whatever the second question answers.
member?
(member? a lat) is just another way to ask "Is a is one of the atoms of the lat".
