石子合并

有 \(n\) 堆石子排成一排，其编号为 \(1, 2, 3, \dots, n\)，每堆石子有一定的质量，可以用一个整数表示，要将这 \(n\) 堆石子合并为一堆，每次合并只能合并相邻的的两堆，合并的代价为这两堆石子的质量之和，合并后与这两堆石子相邻的石子将和新堆相邻，合并时由于选择顺序不同，合并的总代价也不同，计算最小的合并为一堆的代价。

状态表示：\(dp[i][j]\) 表示将区间 \([i, j]\) 内的石子合并为一堆的最小代价。

状态转移方程：\(dp[i][j] = dp[i][k] + dp[k + 1][j] + (preS[j] - preS[i - 1])\)。

\(preS[]\) 是 前缀和。

如果要将 \(i \sim j\) 的石子合并，无论期间的合并过程是什么样的，最后一次合并一定是合并左右两堆的石子。

|--------------|-----------------|
left_stone             right_stone

而且左右两堆的合并互不影响，那么我们就可以枚举 \(k\) 从 \(i + 1\) 到 \(j - 1\)，计算出代价最小时 \(k\) 的值。

#include <cstdio>
#include <algorithm>
using namespace std;
const int maxn = 300 + 5;
int preS[maxn], dp[maxn][maxn];
const int INF = 0x3f3f3f3f;

int main() {
    int n; scanf("%d", &n);
    for (int i = 1; i <= n; ++ i) {
        scanf("%d", &preS[i]);
        preS[i] += preS[i - 1];
    }
    for (int len = 2; len <= n; ++ len)
        for (int i = 1; i + len - 1 <= n; ++ i) {
            int j = i + len - 1;
            dp[i][j] = INF;
            for (int k = i; k < j; ++ k)
                dp[i][j] = min(dp[i][j], dp[i][k] + dp[k + 1][j] + preS[j] - preS[i - 1]);
        }
    printf("%d\n", dp[1][n]);
    return 0;
}

最长回文字串

  • dp[i][j]表示str[i]至str[j]所表示的子串是否是回文子串，是则为1，不是为0，根据str[i]是否等于str[j]
    • 若str[i] == str[j]，那么只要str[i+1]至str[j-1]是回文子串，str[i]至str[j]就是回文子串，如果str[i+1]至str[j-1]不是回文子串，则str[i]至str[j]也不是回文子串
    • 若str[i] != str[j]，那么str[i]至str[j]一定不是回文子串
    • 动态转移方程：
      • dp[i][j] = dp[i+1][j-1] (str[i] == str[j])
      • dp[i][j] = 0 (str[i] != str[j])
      • 边界dp[i][i] = 1; dp[i][i+1] = (str[i] == str[j])?1:0
      • 但是如果按照i和j从小到大的顺序来枚举子串的两个端点，然后更新dp[i][j]，会无法保证dp[i+1][j-1]已经被计算过，从而无法得到正确的dp[i][j]
    • 所以这种方法不行，观察看到转移方程按子串的长度逐渐减少，所以考虑按子串的长度和子串的初始位置进行枚举
